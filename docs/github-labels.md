# Suggested GitHub labels

## Type

- `type:bug` тАФ broken behavior, regressions, crashes
- `type:feature` тАФ new user-visible capability
- `type:docs` тАФ documentation-only work
- `type:refactor` тАФ internal cleanup without new behavior
- `type:infra` тАФ packaging, CI, release, deployment machinery

## Area

- `area:cli` тАФ command-line interface
- `area:ui` тАФ browser UI and front-end behavior
- `area:runtime` тАФ detached/supervisor process handling
- `area:metrics` тАФ event/metric parsing and charts
- `area:docs` тАФ docs site and README
- `area:packaging` тАФ package metadata, install paths, release prep

## Priority

- `prio:p0` тАФ blocks real use
- `prio:p1` тАФ should happen soon
- `prio:p2` тАФ useful but not urgent
- `prio:p3` тАФ nice-to-have / backlog

## Status

- `status:needs-triage`
- `status:ready`
- `status:blocked`
- `status:help-wanted`
- `status:good-first-issue`

## Example combinations

- `type:bug` + `area:runtime` + `prio:p0`
- `type:feature` + `area:ui` + `prio:p1`
- `type:docs` + `area:docs` + `status:good-first-issue`
